# Multicore — LLM Context Summary

## What is this?

Multicore is an open-source Next.js 16 dashboard and server-side automation toolkit for managing
multiple accounts in Terracore, a blockchain game on the Hive network. Players earn SCRAP tokens
by claiming their mining stash, battling other players, completing quests, staking SCRAP, and
upgrading stats (DMG / DEF / ENG). Relics are NFT-style items traded on Terracore's marketplace.

## Tech Stack

- Next.js 16, React 19, TypeScript, Tailwind CSS v4, shadcn/ui
- Hive Keychain (browser extension) for all UI on-chain actions
- @hiveio/dhive for server-side Hive blockchain broadcasting
- CryptoJS (AES-256) for account key encryption at rest
- tsx for running server-side automation scripts directly

## Automation Scripts

All automation scripts run headlessly via `tsx`. They require `.env` to be configured
with `TERRACORE_ACCOUNTS_ENC` and `TERRACORE_ACCOUNT_MAIN` before first run.
Behaviour is controlled by `server/automation/<name>/config/settings.ts`.

| Command                  | What it does                                                   |
|--------------------------|----------------------------------------------------------------|
| pnpm auto:claim-battle   | Attack targets + claim SCRAP stash per account                 |
| pnpm auto:quest          | Collect completed quests + start new quests per account        |
| pnpm auto:token-transfer | Sweep a token (HIVE / HBD / SCRAP / any HE token) to main     |
| pnpm auto:relic-market   | Sub-accounts sell relics; main account buys them back          |
| pnpm auto:terracore      | All of the above combined in one continuous loop (recommended) |

NOTE: Boss fights and stat upgrades are NOT automated. The automation loop covers
claim, attack, quest, and token transfer only.

## Key Environment Variables

```
TERRACORE_ACCOUNTS_ENC       # AES-encrypted account list (generated by dashboard Encrypt tool)
TERRACORE_ENCRYPTION_KEY     # 64-char hex key matching the encrypted config above
TERRACORE_ACCOUNT_MAIN       # Main / receiver account (sweep target + relic buyer)
POLL_INTERVAL                # Loop interval in ms (default 60000)
```

Optional runtime overrides for auto-token-transfer:
```
TOKEN_SYMBOL     SEND_MAX_BALANCE     CUSTOM_AMOUNT     RUN_ONCE
```

## Hive Blockchain Notes

- All on-chain actions use custom_json broadcasts on the Hive blockchain
- Terracore game actions use custom_json id: terracore_claim, terracore_battle,
  terracore_quest_collect
- Hive Engine (token) actions use custom_json id: ssc-mainnet-hive
- Relic marketplace actions use: custom_json id=ssc-mainnet-hive, contractAction=tm_create (sell)
  and requestTransfer to terracore.market (buy)
- Posting key: claim, battle, quest collect, RC delegation
- Active key: upgrade stats, stake SCRAP, burn SCRAP, sell relics, buy relics, token transfer

## Terracore API Base

All game data is fetched from the Terracore REST API. Relevant fetchers live in
`lib/shared/api/terracore.ts`:
- fetchPlayer(username)        → player stats, level, XP, minerate
- fetchPlayerRelics(username)  → relic inventory
- fetchQuestBoard(username)    → available quest slots
- fetchActiveQuests(username)  → in-progress quests
- fetchBattleTargets(maxDef)   → attack targets filtered by defense
